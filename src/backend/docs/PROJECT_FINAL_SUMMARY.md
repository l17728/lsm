## 一、项目概述

### 1.1 项目背景

LSM（实验室服务器资源管理系统）是一个面向实验室环境的 GPU 服务器资源管理平台，旨在解决大规模异构服务器集群的管理难题，实现 GPU 资源的精细化调度与分配。

### 1.2 项目目标

| 目标领域 | 具体目标 | 达成状态 |
|---------|---------|---------|
| 资源管理 | 统一管理 ~2000 台异构服务器 | ✅ 完成 |
| GPU 调度 | 细粒度管理每台服务器的多张 GPU 卡 | ✅ 完成 |
| 可视化 | 直观的资源状态和使用情况展示 | ✅ 完成 |
| 自动化 | 资源分配、回收、任务排期的自动化 | ✅ 完成 |
| 智能运维 | 自动扩缩容、故障自愈、告警降噪 | ✅ 完成 |

### 6.2 踩坑教训

- **TypeScript 编译问题** → 前期重视类型设计，统一类型管理
- **Prisma Schema 同步** → Schema 变更同步更新代码
- **测试 Mock 策略** → 优先集成测试，合理使用 Mock
- **Docker 环境问题** → 尽早使用 Docker 开发环境
- **时间估算偏差** → 预留缓冲时间，风险提前识别
