# LSM v3.2.0 审批流程设计文档

**版本**: 3.2.0  
**创建日期**: 2026-03-15  
**状态**: 设计稿  

---

## 1. 概述

本文档定义 LSM（实验室服务器管理系统）v3.2.0 版本的审批流程设计，采用三层决策机制，平衡系统自动化效率与操作安全性。

---

## 2. 决策分级标准

### 2.1 分级原则

| 级别 | 标识 | 决策方式 | 响应时效 | 适用场景 |
|------|------|----------|----------|----------|
| 🟢 自动执行 | AUTO | Agent 直接执行 | 即时 | 低风险、可逆操作 |
| 🟡 通知确认 | NOTIFY | 执行后通知用户 | 5分钟内 | 中等风险、可追溯操作 |
| 🔴 人工审批 | APPROVAL | 需人工批准 | 依赖审批人 | 高风险、不可逆操作 |

### 2.2 风险评估维度

- **数据影响**: 是否涉及数据删除/修改
- **资源影响**: 影响范围（单机/多机/集群）
- **可逆性**: 操作是否可撤销
- **成本影响**: 是否产生额外费用
- **安全影响**: 是否涉及权限变更/安全配置

## 3. 操作分类表

### 3.1 🟢 自动执行（低风险）

| 操作类型 | 具体操作 | 风险说明 |
|----------|----------|----------|
| 查询类 | 资源状态查询、日志查看、监控数据获取 | 只读操作，无副作用 |
| 信息展示 | 服务器列表、GPU 使用率、任务队列 | 数据展示，不修改状态 |
| 个人设置 | 用户偏好设置、通知配置、界面布局 | 仅影响当前用户 |
| 低影响修改 | 任务描述更新、标签添加、备注修改 | 不影响资源分配 |

### 3.2 🟡 通知确认（中等风险）

| 操作类型 | 具体操作 | 风险说明 |
|----------|----------|----------|
| 资源申请 | GPU 服务器申请、资源预留 | 占用资源，需通知管理员 |
| 任务调度 | 任务提交、优先级调整 | 影响队列，需相关方知晓 |
| 配置变更 | 环境变量修改、容器配置调整 | 可能影响任务运行 |
| 批量操作 | 批量状态更新、批量通知发送 | 影响范围较广 |
| 权限申请 | 项目加入申请、资源访问申请 | 涉及权限，需留痕 |

### 3.3 🔴 人工审批（高风险）

| 操作类型 | 具体操作 | 风险说明 |
|----------|----------|----------|
| 资源释放 | 强制释放他人占用的资源 | 不可逆，影响他人任务 |
| 权限变更 | 用户角色变更、权限授予/收回 | 安全风险，需审核 |
| 系统配置 | 集群配置修改、调度策略调整 | 影响全局，需审批 |
| 数据操作 | 数据删除、备份清理、日志清除 | 数据丢失风险 |
| 费用相关 | 资源扩容、付费功能启用 | 成本影响，需授权 |

## 5. 超时处理机制

### 5.1 审批超时规则

| 审批级别 | 超时时间 | 超时动作 | 通知对象 |
|----------|----------|----------|----------|
| 普通审批 | 24小时 | 自动升级至上级 | 审批人 + 上级管理员 |
| 紧急审批 | 4小时 | 自动升级至上级 | 审批人 + 上级管理员 |
| 关键审批 | 48小时 | 保持待审状态，每日提醒 | 审批人 |

### 5.2 超时处理流程

```
审批请求 ──> 计时开始 ──> 达到50%时间? ──> 发送提醒通知
                                │
                                ▼
                        达到超时阈值?
                           │      │
                          是      否
                           ▼       │
                     自动升级       │
                           ▼       │
                    通知上级管理员   │
                           ▼       │
                     重置计时器 <────┘
```

### 5.3 超时通知模板

```
【审批超时提醒】
请求ID: {request_id}
操作类型: {operation_type}
申请人: {applicant}
等待时间: {waiting_hours}小时
当前状态: 已升级至 {escalation_target}

请尽快处理或转交相关人员。
```

## 6. 紧急操作处理

### 6.1 紧急操作定义

- 集群安全威胁（入侵检测、异常流量）
- 服务宕机影响范围 > 50% 节点
- 数据丢失风险（存储故障、备份失败）
- 合规紧急响应（监管要求、审计整改）

### 6.2 紧急操作流程

```
┌────────────────────────────────────────────────────────────┐
│                     紧急操作触发                            │
└─────────────────────────────┬──────────────────────────────┘
                              ▼
                    ┌─────────────────────┐
                    │  系统自动评估风险    │
                    └──────────┬──────────┘
                              ▼
               ┌──────────────────────────────┐
               │  发送紧急通知至值班管理员      │
               │  (短信 + 邮件 + 系统推送)     │
               └──────────────┬───────────────┘
                              ▼
              ┌───────────────────────────────┐
              │  管理员 15 分钟内响应?          │
              └───────┬───────────────┬────────┘
                     是              否
                      ▼               ▼
              ┌──────────────┐  ┌──────────────────┐
              │ 获得授权后    │  │ 系统自动执行     │
              │ 执行操作      │  │ 预设安全措施     │
              └──────────────┘  └──────────────────┘
```

### 6.3 紧急操作权限矩阵

| 角色 | 单节点操作 | 多节点操作 | 集群级操作 |
|------|-----------|-----------|-----------|
| 普通用户 | ❌ | ❌ | ❌ |
| 项目管理员 | 🟡 通知 | ❌ | ❌ |
| 运维工程师 | 🟢 自动 | 🟡 通知 | 🔴 审批 |
| 系统管理员 | 🟢 自动 | 🟢 自动 | 🟡 通知 |
| 超级管理员 | 🟢 自动 | 🟢 自动 | 🟢 自动 |

## 7. 审批记录与审计

### 7.1 记录内容

- 操作类型、申请人、审批人
- 申请时间、审批时间、执行时间
- 操作详情、审批意见
- 执行结果、影响范围

### 7.2 审计要求

- 审批记录保留至少 1 年
- 支持按时间、操作类型、申请人筛选
- 支持导出审计报表
- 异常审批自动标记并告警
