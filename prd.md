# 实验室服务器管理系统 (LSM) - 产品需求文档

**版本**: 1.0  
**创建日期**: 2026-03-12  
**文档状态**: 初稿  

---

## 1. 项目概述

### 1.1 项目背景

随着人工智能和大模型研究的快速发展，实验室需要管理规模达 2000 台的异构服务器集群，每台服务器配置多张 GPU 卡。当前缺乏统一的资源管理平台，导致资源利用率低、分配混乱、任务排队无序等问题。

### 1.2 项目目标

构建一套实验室服务器管理系统（Laboratory Server Management, LSM），实现：
- 统一纳管 2000+ 台异构服务器及 GPU 资源
- 支持推理平台开发测试、大模型使用、长周期训练任务等多种场景
- 提供可视化资源监控和智能调度能力
- 实现资源自动回收和高效利用

### 1.3 适用范围

本系统适用于高校实验室、科研院所、企业研发中心等需要大规模 GPU 集群管理的场景。

## 2. 竞品调研分析

### 2.1 OpenStack

**定位**: 开源云计算平台，提供 IaaS 层资源管理

**核心功能**:
- 计算资源管理（Nova）：虚拟机生命周期管理
- 存储管理（Cinder/Swift）：块存储和对象存储
- 网络管理（Neutron）：软件定义网络
- 镜像管理（Glance）：虚拟机镜像仓库
- 身份认证（Keystone）：统一认证授权

**优势**:
- 功能全面，覆盖云计算全栈
- 开源生态成熟，社区活跃
- 支持多租户和资源隔离
- 可扩展性强，支持大规模部署

**劣势**:
- 部署复杂，运维成本高
- 对 GPU 资源调度支持有限
- 不适合批处理和高性能计算场景
- 学习曲线陡峭

**适用性评估**: ⭐⭐⭐ (3/5)
- 适合通用云资源管理，但 GPU 调度能力不足

---

### 2.2 Kubernetes + Volcano

**定位**: 容器编排 + 批处理调度系统

**核心功能**:
- 容器编排：Pod 调度、服务发现、自动扩缩容
- Volcano 批处理：队列管理、任务优先级、Gang Scheduling
- GPU 共享：时间片/显存隔离、MIG 支持
- 资源配额：Namespace 级别资源限制
- 弹性调度：基于负载的动态调整

**优势**:
- 云原生架构，生态丰富
- Volcano 专为 AI/ML 工作负载设计
- 支持 GPU 细粒度调度和共享
- 与主流 ML 框架集成良好

**劣势**:
- K8s 本身复杂度高
- 对传统 HPC 工作负载支持有限
- 需要容器化改造现有应用
- 长周期任务管理不够完善

**适用性评估**: ⭐⭐⭐⭐ (4/5)
- 适合容器化 AI 工作负载，是重要参考方向

---

### 2.3 Slurm (Simple Linux Utility for Resource Management)

**定位**: 高性能计算工作负载管理器

**核心功能**:
- 资源发现和管理：节点、CPU、GPU、内存
- 作业调度：优先级队列、公平共享、抢占式调度
- 作业管理：提交、监控、取消、重新排队
- 资源预留：未来时间段的资源锁定
- 计费统计：资源使用量统计和报表

**优势**:
- HPC 领域事实标准，成熟稳定
- 支持超大规模集群（万级节点）
- 调度算法丰富，支持多种策略
- 对长周期训练任务支持好
- 轻量级部署，运维简单

**劣势**:
- 用户界面较原始（命令行为主）
- 缺乏容器原生支持
- 可视化能力弱
- 对云原生生态集成有限

**适用性评估**: ⭐⭐⭐⭐⭐ (5/5)
- 最接近目标场景，是核心参考对象

---

### 2.4 Run:AI

**定位**: GPU 虚拟化和池化平台，专注机器学习

**核心功能**:
- GPU 虚拟化：单卡多任务、显存隔离
- 资源池化：跨节点 GPU 统一调度
- 策略引擎：基于策略的自动调度
- 监控分析：GPU 利用率、任务性能分析
- 集成能力：与 K8s、Slurm 等集成

**优势**:
- GPU 利用率显著提升（3-5 倍）
- 商业产品，有专业支持
- 可视化界面友好
- 针对 ML 工作负载优化

**劣势**:
- 商业软件，成本高
- 闭源，定制能力有限
- 依赖现有基础设施（K8s/Slurm）
- 国内支持有限

**适用性评估**: ⭐⭐⭐⭐ (4/5)
- 功能理念先进，值得借鉴其资源池化和虚拟化思路

---

### 2.5 竞品对比总结

| 特性 | OpenStack | K8s+Volcano | Slurm | Run:AI | LSM 目标 |
|------|-----------|-------------|-------|--------|----------|
| 规模支持 | 万级 | 千级 | 万级 | 千级 | 2000+ 节点 |
| GPU 调度 | 弱 | 强 | 强 | 极强 | 强 |
| 批处理支持 | 弱 | 中 | 强 | 中 | 强 |
| 可视化 | 中 | 中 | 弱 | 强 | 强 |
| 部署复杂度 | 高 | 高 | 低 | 中 | 中 |
| 开源程度 | 开源 | 开源 | 开源 | 闭源 | 开源 |
| 长周期任务 | 弱 | 中 | 强 | 中 | 强 |

**LSM 设计原则**:
- 借鉴 Slurm 的作业调度和资源管理核心
- 吸收 K8s+Volcano 的云原生理念
- 学习 Run:AI 的可视化与资源池化思想
- 保持轻量级部署和易运维特性

## 3. 用户角色与权限体系

### 3.1 用户角色定义

#### 3.1.1 系统管理员 (System Admin)
- **描述**: 负责整个平台的运维和管理
- **典型用户**: 实验室网络管理员、IT 运维人员
- **权限范围**: 全系统

#### 3.1.2 资源管理员 (Resource Admin)
- **描述**: 负责资源分配、审批和配额管理
- **典型用户**: 实验室负责人、课题组 PI
- **权限范围**: 指定资源池

#### 3.1.3 高级研究员 (Senior Researcher)
- **描述**: 有较高资源配额的科研人员
- **典型用户**: 教授、博士后、高级工程师
- **权限范围**: 个人配额 + 项目组资源

#### 3.1.4 研究员/学生 (Researcher/Student)
- **描述**: 普通资源使用者
- **典型用户**: 研究生、本科生、初级研究员
- **权限范围**: 个人配额

#### 3.1.5 访客 (Guest)
- **描述**: 临时访问人员
- **典型用户**: 访问学者、外部合作人员
- **权限范围**: 受限资源，需审批

### 3.2 权限体系

```
权限层级:
├── 系统级权限
│   ├── 用户管理 (创建/删除/修改用户)
│   ├── 角色管理 (定义角色和权限)
│   ├── 系统配置 (全局参数设置)
│   └── 审计日志 (查看全系统操作日志)
│
├── 资源级权限
│   ├── 节点管理 (添加/移除服务器)
│   ├── GPU 管理 (配置 GPU 属性)
│   ├── 资源池管理 (创建/修改资源池)
│   └── 配额管理 (设置用户/组配额)
│
├── 作业级权限
│   ├── 作业提交 (提交新任务)
│   ├── 作业管理 (查看/取消/重启作业)
│   ├── 优先级调整 (修改作业优先级)
│   └── 资源预留 (预约未来资源)
│
└── 数据级权限
    ├── 监控数据 (查看资源使用统计)
    ├── 报表导出 (生成使用报告)
    └── 告警配置 (设置监控告警)
```

## 4. 核心功能清单

### 4.1 资源管理 (Resource Management)

#### 4.1.1 服务器纳管
- 自动发现网络中的服务器节点
- 支持手动添加/移除节点
- 节点信息登记（主机名、IP、配置、位置）
- 节点状态监控（在线/离线/维护）
- 节点标签系统（按用途、归属、特性分类）

#### 4.1.2 GPU 资源管理
- GPU 自动发现（型号、显存、状态）
- GPU 健康检查（温度、功耗、错误计数）
- GPU 隔离配置（独占/共享模式）
- GPU 属性标记（A100/H100/4090 等）
- 多卡拓扑感知（NVLink、PCIe 带宽）

#### 4.1.3 资源池管理
- 动态资源池创建（按项目组、用途划分）
- 资源池配额设置（GPU 数量、时间配额）
- 资源池隔离策略（独占/共享）
- 资源池使用统计和报表

#### 4.1.4 异构资源支持
- 不同 GPU 型号混部管理
- CPU/内存/存储资源统筹
- 网络资源管理（带宽、RDMA）
- 特殊硬件支持（TPU、FPGA 等）

---

### 4.2 可视化大屏 (Dashboard)

#### 4.2.1 全局概览
- 集群资源总览（总 GPU 数、可用数、使用率）
- 实时任务统计（运行中、排队中、已完成）
- 资源利用率趋势图（24h/7d/30d）
- 告警信息滚动展示

#### 4.2.2 资源视图
- 节点拓扑图（物理位置、网络关系）
- GPU 状态热力图（按使用率/温度着色）
- 资源池使用对比图
- 用户/组资源占用排名

#### 4.2.3 任务视图
- 任务甘特图（时间轴展示任务排布）
- 任务队列状态（等待时间预估）
- 任务详情钻取（日志、指标、资源占用）
- 任务历史查询和对比

#### 4.2.4 自定义大屏
- 可配置的组件布局
- 多维度数据筛选
- 自动刷新和告警联动
- 支持多屏展示（不同角色视图）

---

### 4.3 资源分配 (Resource Allocation)

#### 4.3.1 配额管理
- 用户基础配额（GPU 数量、时长）
- 组/项目共享配额
- 弹性配额（空闲时借用）
- 配额使用统计和预警

#### 4.3.2 分配策略
- 公平共享（Fair Share）调度
- 优先级队列（高优先级优先）
- 先到先得（FCFS）模式
- 基于贡献度分配（论文/项目产出）

#### 4.3.3 动态分配
- 空闲资源自动分配
- 低优先级任务抢占机制
- 资源碎片整理和合并
- 跨资源池借用审批流程

#### 4.3.4 预留机制
- 未来时间段资源预约
- 周期性预留（每周固定时段）
- 预留冲突检测和协调
- 预留超时自动释放

---

### 4.4 自动回收 (Auto Reclamation)

#### 4.4.1 空闲检测
- GPU 利用率监控（连续低负载检测）
- 任务停滞检测（无进度更新）
- 会话超时检测（无操作超时）
- 异常进程检测（僵尸进程）

#### 4.4.2 回收策略
- 分级告警（预警→警告→强制回收）
- 宽限期设置（给用户缓冲时间）
- 自动保存检查点（防止数据丢失）
- 回收通知（邮件/短信/站内信）

#### 4.4.3 例外处理
- 白名单机制（关键任务豁免）
- 延期申请流程（需审批）
- 手动保活接口（API/界面）
- 回收申诉渠道

#### 4.4.4 回收执行
- 优雅终止（发送 SIGTERM）
- 强制终止（SIGKILL，超时后）
- 资源释放确认
- 回收日志记录

---

### 4.5 工作计划排布 (Job Scheduling)

#### 4.5.1 作业提交
- Web 界面提交（表单/拖拽）
- 命令行提交（兼容 Slurm 语法）
- API 提交（RESTful 接口）
- 作业模板（常用配置保存）

#### 4.5.2 调度策略
- 优先级调度（用户优先级 + 作业优先级）
- 资源匹配（GPU 型号/数量/拓扑）
- 装箱优化（减少碎片）
- 公平性保障（防止饥饿）

#### 4.5.3 队列管理
- 多队列支持（开发/训练/推理）
- 队列配额限制
- 队列优先级设置
- 队列间借用机制

#### 4.5.4 作业生命周期
- 待调度 → 运行中 → 已完成/失败/取消
- 作业暂停/恢复
- 作业重启（从检查点）
- 作业依赖（DAG 工作流）

#### 4.5.5 长周期任务支持
- 检查点自动保存
- 断点续训支持
- 跨节点迁移（故障时）
- 超时保护和延期

---

### 4.6 辅助功能

#### 4.6.1 用户管理
- 用户注册和认证（支持 LDAP/SSO）
- 用户组管理
- 用户配额查看
- 使用历史查询

#### 4.6.2 监控告警
- 资源指标监控（GPU/CPU/内存/网络）
- 告警规则配置
- 多渠道通知（邮件/短信/微信/钉钉）
- 告警升级机制

#### 4.6.3 日志审计
- 操作日志（谁在何时做了什么）
- 作业日志（任务执行记录）
- 系统日志（平台运行状态）
- 日志检索和导出

#### 4.6.4 报表统计
- 资源使用报表（按用户/组/时间）
- 作业统计报表（成功率、等待时间）
- 成本分摊报表（资源使用折算）
- 自定义报表生成

## 5. 用户故事 (User Stories)

### 5.1 管理员视角

#### US-ADMIN-01: 节点批量导入
> 作为系统管理员，我希望能够批量导入服务器节点信息，以便快速完成 2000 台服务器的初始化纳管。

**验收标准**:
- 支持 CSV/Excel 格式批量导入
- 导入前进行数据校验
- 导入结果反馈（成功/失败明细）
- 支持导入回滚

#### US-ADMIN-02: 资源配额调整
> 作为资源管理员，我希望能够动态调整用户/组的资源配额，以便根据项目进展灵活分配资源。

**验收标准**:
- 支持实时调整配额（立即生效）
- 支持定时配额（特定时间段生效）
- 调整操作需要审计日志
- 配额变更通知相关用户

#### US-ADMIN-03: 异常节点处理
> 作为系统管理员，我希望能够快速定位和处理异常节点，以便最小化对任务的影响。

**验收标准**:
- 异常节点自动告警
- 节点健康诊断工具
- 一键隔离异常节点
- 节点维护模式（暂停调度）

---

### 5.2 研究员视角

#### US-RESEARCHER-01: 快速提交训练任务
> 作为高级研究员，我希望能够快速提交大模型训练任务，并指定所需的 GPU 型号和数量，以便开展研究工作。

**验收标准**:
- 支持选择 GPU 型号（A100/H100 等）
- 支持指定 GPU 数量（1-8 卡）
- 支持多节点分布式任务
- 提交后显示预计等待时间

#### US-RESEARCHER-02: 任务进度监控
> 作为研究员，我希望能够实时查看我的任务执行进度和资源使用情况，以便及时调整训练策略。

**验收标准**:
- 实时 GPU 利用率展示
- 训练日志实时查看
- 预计完成时间估算
- 支持移动端查看

#### US-RESEARCHER-03: 资源预约
> 作为研究员，我希望能够预约未来时间段的资源，以便确保重要实验能够按时进行。

**验收标准**:
- 支持预约未来 7 天内资源
- 预约冲突自动检测
- 预约审批流程（如需）
- 预约到期提醒

---

### 5.3 学生视角

#### US-STUDENT-01: 查看可用资源
> 作为学生，我希望能够查看当前可用的资源情况，以便选择合适的时段提交任务。

**验收标准**:
- 实时显示各资源池可用 GPU 数
- 显示预计等待时间
- 按 GPU 型号筛选
- 低峰时段推荐

#### US-STUDENT-02: 开发调试任务
> 作为学生，我希望能够提交短时的开发调试任务，并快速获取资源，以便进行代码调试。

**验收标准**:
- 开发队列快速通道
- 支持 1 卡短时任务（<4 小时）
- 自动回收闲置资源
- 调试环境预配置

#### US-STUDENT-03: 配额使用查询
