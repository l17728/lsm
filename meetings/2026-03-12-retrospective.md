## ⚠️ 经验教训复盘

### 教训 1: Express 5 兼容性问题

**问题**: 服务启动失败，路由语法错误

**原因**: Express 5 使用新版 path-to-regexp，不支持 `/:path(*)` 语法

**解决**: 改用查询参数方式 `/view?file=path`

**教训**:
1. ✅ 升级依赖前先查看变更日志
2. ✅ 在测试环境验证后再部署
3. ✅ 记录版本兼容性问题

**改进措施**:
- 建立依赖升级检查清单
- 记录已知兼容性问题
- 创建技术债务清单

---

### 教训 2: 服务自启动配置延迟

**问题**: 手动启动服务重启后丢失

**原因**: 未及时配置 systemd 服务

**解决**: 创建 systemd 服务文件，配置开机自启

**教训**:
1. ✅ 生产服务必须配置自启动
2. ✅ 使用 systemd 管理服务
3. ✅ 配置日志记录

**改进措施**:
- 创建服务部署检查清单
- 包含 systemd 配置步骤
- 添加服务监控

---

### 教训 3: Git 版本控制滞后

**问题**: 代码未及时纳入版本管理

**原因**: 项目开始未立即初始化 Git

**解决**: 立即初始化并提交所有代码

**教训**:
1. ✅ 项目开始先初始化 Git
2. ✅ 及时提交代码
3. ✅ 编写规范的提交信息

**改进措施**:

- 配置 Git hooks
