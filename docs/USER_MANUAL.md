## 系统概述

### 什么是 LSM？

LSM (Laboratory Server Management System) 是一个全面的实验室服务器资源管理平台，专为科研实验室设计，用于高效管理服务器资源、GPU 分配和任务调度。

### 核心功能

| 功能模块 | 说明 |
|---------|------|
| 🖥️ 服务器管理 | 管理多台服务器，实时跟踪状态 |
| 🎮 GPU 分配 | 动态 GPU 资源分配与回收 |
| 📋 任务调度 | 基于优先级的任务调度系统 |
| 📊 实时监控 | WebSocket 实时更新监控数据 |
| 🔐 安全认证 | JWT 认证、双因素认证 (2FA)、审计日志 |
| 📧 邮件通知 | 事件驱动的邮件提醒系统 |
| 📱 移动端适配 | 响应式设计，支持移动设备 |
| 📤 数据导出 | 支持 CSV/Excel 格式导出 |

### 技术架构

### 用户角色

| 角色 | 权限 |
|------|------|
| **ADMIN** | 所有权限，包括用户管理、服务器管理 |
| **MANAGER** | 服务器和任务管理，不能管理用户 |
| **USER** | 基本使用权限，管理自己的任务 |

## 快速入门

### 1. 访问系统

打开浏览器，访问系统地址：
- **开发环境**: http://localhost:3000
- **生产环境**: https://your-domain.com

### 2. 注册账号

1. 点击"注册"按钮
2. 填写以下信息：
   - 用户名：3-30 个字符
   - 邮箱：有效的邮箱地址
   - 密码：至少 6 个字符
3. 点击"注册"完成账号创建

### 3. 登录系统

1. 输入用户名和密码
2. 点击"登录"
3. (可选) 完成双因素认证

### 4. 首次使用

#### 4.1 添加服务器 (管理员)

1. 导航到左侧菜单"服务器管理"
2. 点击"添加服务器"按钮
3. 填写服务器信息：
   - **名称**: 例如 "GPU-Server-01"
   - **主机名**: 例如 "gpu01.lab.local"
   - **IP 地址**: 例如 "192.168.1.100"
   - **CPU 核心数**: 例如 32
   - **总内存 (GB)**: 例如 128
   - **GPU 数量**: 例如 4
4. 点击"确定"保存

#### 4.2 分配 GPU

1. 导航到"GPU 管理"
2. 点击"分配 GPU"按钮
3. 选择筛选条件（可选）：
   - 显存大小
   - GPU 型号
4. 点击"确认分配"

#### 4.3 创建任务

1. 导航到"任务管理"
2. 点击"创建任务"按钮
3. 填写任务信息：
   - **任务名称**: 例如 "模型训练"
   - **描述**: 任务详细描述（可选）
   - **优先级**: 0-10（越高越优先）
   - **计划执行时间**: 可选
4. 点击"确定"创建任务

## 核心功能使用

### 1. 服务器管理

#### 查看服务器列表

- 导航到"服务器管理"
- 查看所有服务器的状态概览
- 状态标识：
  - 🟢 在线 (ONLINE)
  - 🔴 离线 (OFFLINE)
  - 🟡 维护中 (MAINTENANCE)
  - ⚠️ 错误 (ERROR)

#### 添加服务器

**前提条件**: 需要 ADMIN 角色

1. 点击"添加服务器"按钮
2. 填写必填信息：
   - 名称（必填）
   - 主机名（必填）
   - IP 地址（必填，需为有效 IP 格式）
   - CPU 核心数（必填，≥1）
   - 总内存（必填，≥1 GB）
   - GPU 数量（可选）
3. 点击"确定"保存

#### 编辑服务器

**前提条件**: 需要 MANAGER 或 ADMIN 角色

1. 在服务器列表中找到目标服务器
2. 点击"编辑"按钮
3. 修改需要更新的信息
4. 点击"保存"

#### 更新服务器状态

**前提条件**: 需要 MANAGER 或 ADMIN 角色

1. 找到目标服务器
2. 点击"状态"下拉菜单
3. 选择新状态：
   - ONLINE - 在线
   - OFFLINE - 离线
   - MAINTENANCE - 维护中
   - ERROR - 错误
4. 确认更新

#### 删除服务器

**前提条件**: 需要 ADMIN 角色

⚠️ **警告**: 删除操作不可恢复！

1. 找到目标服务器
2. 点击"删除"按钮
3. 在确认对话框中点击"确认删除"

### 2. GPU 管理

#### 查看 GPU 状态

1. 导航到"GPU 管理"
2. 查看所有 GPU 的实时状态：
   - 是否已分配
   - 分配给用户
   - 显存大小
   - 所属服务器

#### 分配 GPU

1. 点击"分配 GPU"按钮
2. （可选）设置筛选条件：
   - 最小显存
   - GPU 型号
3. 系统自动分配符合条件的可用 GPU
4. 确认分配信息
5. 点击"确认"

#### 释放 GPU

1. 找到已分配的 GPU
2. 点击"释放"按钮
3. 确认释放操作
4. GPU 返回可用池

### 3. 任务管理

#### 查看任务列表

1. 导航到"任务管理"
2. 查看您的所有任务
3. 可筛选状态：
   - 全部
   - 待处理 (PENDING)
   - 运行中 (RUNNING)
   - 已完成 (COMPLETED)
   - 已失败 (FAILED)
   - 已取消 (CANCELLED)

#### 创建任务

1. 点击"创建任务"按钮
2. 填写任务信息：
   - **任务名称**: 1-100 字符
   - **描述**: 详细说明（可选）
   - **优先级**: 0-10，数字越大优先级越高
   - **计划执行时间**: 可选，设定任务开始时间
3. 点击"确定"创建

#### 查看任务详情

1. 在任务列表中点击任务名称
2. 查看详细信息：
   - 任务状态
   - 创建时间
   - 优先级
   - 执行结果（如已完成）

#### 更新任务

1. 找到目标任务
2. 点击"编辑"按钮
3. 修改任务信息
4. 点击"保存"

#### 取消任务

**前提条件**: 只能取消自己创建的任务

1. 找到目标任务
2. 点击"取消"按钮
3. 确认取消操作

⚠️ **注意**: 运行中的任务取消可能需要时间

#### 删除任务

**前提条件**: 只能删除自己创建的任务

1. 找到目标任务
2. 点击"删除"按钮
3. 确认删除操作

### 4. 用户管理 (管理员)

#### 查看用户列表

**前提条件**: 需要 ADMIN 角色

1. 导航到"用户管理"
2. 查看所有用户信息：
   - 用户名
   - 邮箱
   - 角色
   - 注册时间

#### 修改用户角色

**前提条件**: 需要 ADMIN 角色

1. 找到目标用户
2. 点击"编辑角色"按钮
3. 选择新角色：
   - ADMIN - 管理员
   - MANAGER - 管理员
   - USER - 普通用户
4. 确认修改

#### 删除用户

**前提条件**: 需要 ADMIN 角色

⚠️ **警告**: 删除操作不可恢复！

1. 找到目标用户
2. 点击"删除"按钮
3. 在确认对话框中点击"确认删除"

### 5. 监控中心

#### 系统健康状态

1. 导航到"监控中心"
2. 查看系统健康状态：
   - 数据库连接
   - Redis 缓存
   - API 响应时间
   - 内存使用率

#### 性能指标

查看实时性能数据：
- CPU 使用率
- 内存使用率
- 磁盘使用率
- 网络流量

#### 缓存统计

查看缓存性能：

### 6. 数据导出

#### 导出服务器数据

1. 导航到"服务器管理"
2. 点击"导出"按钮
3. 选择格式：
   - CSV

4. 下载文件

## 常见问题解答 (FAQ)

### 账号相关

#### Q1: 忘记密码怎么办？

**A**: 目前系统不支持自助重置密码。请联系管理员重置您的密码。

#### Q2: 如何启用双因素认证 (2FA)？

**A**: 
1. 登录后进入"个人设置"
2. 点击"启用 2FA"
3. 使用认证 APP（如 Google Authenticator）扫描二维码
4. 输入验证码完成绑定

#### Q3: 如何修改密码？

**A**:
1. 登录后进入"个人设置"
2. 点击"修改密码"
3. 输入当前密码和新密码
4. 确认修改

### 服务器管理

#### Q4: 添加服务器时提示"IP 地址格式错误"？

**A**: 请确保输入的是有效的 IPv4 或 IPv6 地址格式，例如：
- ✅ 正确：192.168.1.100
- ❌ 错误：192.168.1.1000（超出范围）
- ❌ 错误：abc.def.ghi.jkl（非数字）

#### Q5: 服务器状态一直显示"离线"？

**A**: 可能原因：
1. 服务器网络连接问题
2. 服务器未启动监控代理
3. 防火墙阻止连接

解决方法：
1. 检查服务器网络连接
2. 确认监控服务运行正常
3. 检查防火墙规则

### GPU 分配

#### Q6: 无法分配 GPU，提示"没有可用 GPU"？

**A**: 可能原因：
1. 所有 GPU 已被分配
2. 没有符合筛选条件的 GPU
3. 服务器状态为离线

解决方法：
1. 释放不需要的 GPU 分配
2. 调整筛选条件
3. 检查服务器状态

#### Q7: 如何查看 GPU 的使用情况？

**A**:
1. 导航到"GPU 管理"
2. 查看每个 GPU 的状态
3. 已分配的 GPU 会显示分配用户和时间

### 任务管理

#### Q8: 任务优先级如何工作？

**A**: 
- 优先级范围：0-10
- 数字越大，优先级越高
- 调度器优先执行高优先级任务
- 相同优先级按创建时间排序

#### Q9: 任务执行失败怎么办？

**A**:
1. 查看任务详情中的错误信息
2. 检查资源配置是否充足
3. 联系管理员查看系统日志
4. 重新创建任务

#### Q10: 如何取消正在运行的任务？

**A**:
1. 找到运行中的任务
2. 点击"取消"按钮
3. 系统会优雅地停止任务
4. 注意：某些任务可能需要时间完成清理

### 系统使用

#### Q11: 页面加载缓慢怎么办？

**A**:
1. 检查网络连接
2. 清除浏览器缓存
3. 尝试刷新页面
4. 如持续缓慢，联系管理员

#### Q12: WebSocket 连接失败？

**A**:
1. 检查后端服务是否运行
2. 查看浏览器控制台错误信息
3. 确认防火墙未阻止 WebSocket
4. 尝试重新登录

#### Q13: 如何获取 API 访问令牌？

**A**:
1. 登录后，令牌会自动存储在本地
2. 如需手动获取，可访问开发者工具
3. 或使用 API 的 `/api/auth/login` 端点

### 邮件通知

#### Q14: 收不到邮件通知？

**A**:
1. 检查邮箱地址是否正确
2. 查看垃圾邮件文件夹
3. 确认邮件服务配置正确
4. 联系管理员检查邮件队列

## 最佳实践

### 1. 服务器管理最佳实践

#### 命名规范
- 使用有意义的名称：`GPU-Server-01` 而非 `Server1`
- 包含位置信息：`BJ-Lab-GPU-01`（北京实验室）
- 保持一致的命名格式

#### 状态管理
- 定期更新服务器状态
- 维护前提前设置为 MAINTENANCE
- 故障时及时标记为 ERROR

#### 监控建议
- 配置告警阈值
- 定期检查性能指标
- 保留历史数据用于分析

### 2. GPU 分配最佳实践

#### 分配策略
- 按需分配，避免占用不用的 GPU
- 任务完成后及时释放
- 使用筛选条件精确匹配需求

#### 资源优化
- 优先使用空闲 GPU
- 合理设置显存筛选条件
- 避免过度分配

### 3. 任务管理最佳实践

#### 任务创建
- 使用描述性的任务名称
- 提供详细的任务描述
- 设置合理的优先级

#### 优先级设置指南
| 优先级 | 使用场景 |
|--------|---------|
| 0-2 | 低优先级，测试任务 |
| 3-5 | 普通优先级，日常任务 |
| 6-8 | 高优先级，重要任务 |
| 9-10 | 紧急任务，生产任务 |

#### 任务执行
- 定期查看任务状态
- 失败任务及时分析原因
- 完成任务及时清理

### 4. 安全最佳实践

#### 账号安全
- 使用强密码（至少 8 位，包含大小写字母、数字、特殊字符）
- 启用双因素认证
- 定期更换密码
- 不共享账号

#### 会话管理
- 离开时注销登录
- 不在公共电脑保存登录状态
- 定期检查登录历史

#### 数据安全
- 敏感数据导出后妥善保管
- 定期备份重要数据
- 遵守数据访问权限

### 5. 性能优化建议

#### 查询优化
- 使用筛选条件减少数据量
- 定期清理历史数据
- 避免频繁刷新

#### 缓存利用
- 系统自动缓存常用数据
- 缓存命中率应保持在 80% 以上
- 数据更新后缓存自动刷新

### 6. 故障排查流程

#### 问题定位
1. 查看错误信息
2. 检查相关组件状态
3. 查看系统日志
4. 重现问题（如可能）

#### 问题解决
1. 尝试标准解决方案
2. 查阅文档和 FAQ
3. 联系技术支持
4. 记录解决方案
