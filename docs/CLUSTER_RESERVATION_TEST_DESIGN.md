### 二、业务流程测试 (Business Flow Tests)

#### UC-BF-001: 完整预约流程（顺利路径）
```
角色: MANAGER
步骤:
  1. 登录系统
  2. 查看集群列表
  3. 选择空闲集群
  4. 填写预约信息（时间、用途）
  5. 提交申请
  6. 等待审批
  7. SUPER_ADMIN 审批通过
  8. 查看我的预约列表
  9. 到达开始时间后自动激活
  10. 使用完毕后释放资源

验证点:
  ✓ 每步操作成功
  ✓ 状态正确流转
  ✓ 集群状态正确更新
  ✓ 日志完整记录

预期状态流转:
  PENDING → APPROVED → ACTIVE → COMPLETED
```

#### UC-BF-002: 预约被拒绝流程
```
步骤:
  1-5. 同上
  6. SUPER_ADMIN 拒绝（填写原因）
  7. 用户收到拒绝通知
  8. 用户重新提交

验证点:
  ✓ 状态变为 REJECTED
  ✓ 显示拒绝原因
  ✓ 可以重新申请
```

#### UC-BF-003: 用户取消预约
```
前置: 预约状态为 PENDING
步骤:
  1. 查看我的预约
  2. 点击取消
  3. 确认取消

验证点:
  ✓ 状态变为 CANCELLED
  ✓ 集群状态恢复为 AVAILABLE
  ✓ 等待队列中的下一个预约推进
```

#### UC-BF-004: 提前释放资源
```
前置: 预约状态为 APPROVED 或 ACTIVE
步骤:
  1. 点击释放资源
  2. 确认释放

验证点:
  ✓ 状态变为 COMPLETED
  ✓ 实际结束时间记录
  ✓ 集群状态恢复
  ✓ 等待队列处理
```

#### UC-BF-005: 进入等待队列
```
前置: 目标时间范围内已有其他预约
步骤:
  1. 申请预约（时间冲突）
  2. 提交申请

验证点:
  ✓ 创建成功，状态为 PENDING
  ✓ 显示队列位置
  ✓ 提示"进入等待队列"
```

#### UC-BF-006: 队列自动推进
```
前置: 队列中有多个等待预约
步骤:
  1. 第一个预约被释放
  2. 系统自动检查队列

验证点:
  ✓ 下一个预约状态更新
  ✓ 队列位置重新计算
  ✓ 用户收到通知
```

#### UC-BF-007: 预约过期自动完成
```
前置: 预约到达结束时间
步骤:
  1. 等待结束时间到达
  2. 定时任务执行

验证点:
  ✓ 状态自动变为 COMPLETED
  ✓ 集群状态恢复
  ✓ 记录实际结束时间
```

#### UC-BF-008: 多集群并发预约
```
步骤:
  1. 同时申请多个不同集群
  2. 查看预约列表

验证点:
  ✓ 所有预约独立处理
  ✓ 各集群状态独立
```

#### UC-BF-009: 日历同步
```
步骤:
  1. 创建预约
  2. 访问日历页面

验证点:
  ✓ 预约信息显示在日历
  ✓ 时间范围正确
  ✓ 集群名称显示
```

## 🤖 AI 智能推荐测试 (AI Recommendation Tests)

### UC-AI-001: 基本推荐功能
```
前置条件: 用户已登录为 MANAGER
步骤:
  1. 打开预约表单
  2. 点击"获取推荐"按钮

验证点:
  ✓ 返回最多 5 个推荐时间段
  ✓ 每个推荐包含 startTime, endTime, score, confidence, reasons
  ✓ 推荐按 score 降序排列
  ✓ score 在 0-100 范围内
  ✓ confidence 在 0-1 范围内
```

### UC-AI-002: 不同时长推荐
```
步骤:
  1. 选择 1 小时
  2. 获取推荐
  3. 选择 4 小时
  4. 获取推荐
  5. 选择 8 小时
  6. 获取推荐

验证点:
  ✓ 每个时长返回适当的推荐
  ✓ 推荐时间段长度与选择时长匹配
  ✓ 不同时长可能返回不同推荐
```

### UC-AI-003: 时间冲突检测
```
前置条件: 存在与请求时间冲突的预约
步骤:
  1. 请求推荐
  2. 检查推荐列表

验证点:
  ✓ 推荐理由中包含冲突提示
  ✓ 或推荐时间段避开冲突
  ✓ queuePosition 显示排队位置
```

### UC-AI-004: 首选时间范围
```
步骤:
  1. 设置 preferredStartTime 为明天 9:00
  2. 设置 preferredEndTime 为明天 18:00
  3. 获取推荐

验证点:
  ✓ 推荐时间在首选范围内
  ✓ 或靠近首选范围
```

### UC-AI-005: 推荐应用功能
```
步骤:
  1. 获取推荐列表
  2. 点击某个推荐

验证点:
  ✓ 时间选择器自动填充推荐时间
  ✓ 开始时间正确
  ✓ 结束时间正确
```

### UC-AI-006: 缺失参数验证
```
步骤:
  1. 不传 clusterId
  2. 不传 duration

验证点:
  ✓ 返回 400 错误
  ✓ 错误信息指明缺失参数
```

### UC-AI-007: 服务错误处理
```
前置条件: clusterId 不存在
```
