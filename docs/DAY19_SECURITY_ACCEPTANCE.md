## 5. 遗留风险

### 5.1 Git 历史泄露 (需处理)

- **风险**: `.env.production` 曾被提交，历史中仍包含真实密钥
- **影响**: 如仓库被克隆，密钥可能泄露
- **建议**: 执行 `git filter-repo` 清理历史，并轮换所有密钥
- **优先级**: 🔴 紧急

### 5.2 依赖漏洞 (待更新)

- **风险**: minimatch ReDoS 漏洞 (高危)
- **影响**: 可能导致服务拒绝
- **建议**: 执行 `npm audit fix`
- **优先级**: 🟠 高

### 5.3 密钥管理服务 (建议集成)

- **风险**: 当前依赖环境变量，未使用专业密钥管理
- **影响**: 密钥轮换和审计较困难
- **建议**: 集成 AWS Secrets Manager 或 Vault
- **优先级**: 🟡 中
