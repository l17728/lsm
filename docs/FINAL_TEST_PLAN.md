## 1. 测试范围和目标

### 1.1 测试范围

#### 在测范围

| 模块 | 范围说明 | 优先级 |
|------|---------|--------|
| 用户认证模块 | 用户注册、登录、权限管理、双因素认证、会话管理 | P0 |
| 服务器管理模块 | 服务器增删改查、状态监控、批量操作、节点管理 | P0 |
| GPU 管理模块 | GPU 分配、释放、状态监控、健康检查 | P0 |
| 任务调度模块 | 任务提交、调度、执行、监控、取消 | P0 |
| 监控模块 | 指标采集、告警规则、数据可视化、报表导出 | P1 |
| 自动扩缩容 (v3.1.0) | 响应式/预测性/定时扩缩容策略、手动扩缩容 | P1 |
| 故障自愈 (v3.1.0) | 故障检测、自动修复、修复历史、人工确认机制 | P1 |
| 告警降噪 (v3.1.0) | 告警去重、聚合、抑制、静默规则 | P1 |

#### 不在测范围

- 第三方依赖库的内部实现
- 操作系统层面的安全漏洞
- 硬件故障模拟（仅做软件层面模拟）
- 网络设备配置

### 1.2 测试目标

| 目标类型 | 具体目标 | 验收标准 |
|---------|---------|---------|
| 功能完整性 | 所有功能按需求规格正常工作 | 100% 测试用例通过 |
| 性能达标 | 系统在 2000 节点规模下稳定运行 | API 响应 < 200ms，错误率 < 1% |
| 安全合规 | 无高危安全漏洞 | 安全评分 ≥ 90 分 |
| 兼容性 | 支持主流浏览器和移动设备 | 100% 兼容性测试通过 |
| 可靠性 | 系统可用性达到生产标准 | 可用性 ≥ 99.9% |
