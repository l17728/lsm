## 🔧 技术亮点

### 1. WebSocket 通知架构
```
业务层 → WebSocket 推送 → 异步保存 → 数据库
           ↓
        用户接收 (实时)
```

- 推送与持久化分离
- 异步保存不阻塞
- 20+ 通知类型全覆盖

### 2. 读写分离路由
```typescript
写操作 → 主库 (强一致)
读操作 → 从库 (轮询负载均衡)
关键读 → 主库 (强一致)
事务 → 主库 (ACID)
```

- 智能路由策略
- 复制延迟监控
- 故障自动降级

### 3. Redis Streams 队列
```
生产者 → Stream → 消费者组 → 处理
                  ↓
              死信队列 (失败)
```

- 持久化可靠
- 消费者组并发
- 自动重试机制

### 4. 高可用设计
```
多层降级:
1. Redis 不可用 → 内存队列
2. 从库不可用 → 主库读取
3. 主库不可用 → 从库提升
4. 实例故障 → 负载均衡剔除
```

### 5. 水平扩展架构
```
Nginx (负载均衡)
  ↓
Backend-1, Backend-2, Backend-3
  ↓
Redis (会话共享) + DB (主从复制)
```
